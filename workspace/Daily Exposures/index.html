What is Version Control?

Version control enables you to work on a project within a team without overwritting each others previous work. This helps keep everybody working on the project on the same page. Version control also helps keep your work backed up and gives you the ability to revert back to previous commits if you ever run into a problem or need to for any reason. Version control solves the file sharing problem!

What is Git?

Git is an open source distributed version control program. Git can be used for large and small projects. Git allows you to have multiple workflows and local repositories. Git is used by many fortune 500 companies on their projects! Every user on Git has a complete copy of the repository stored on their machine. This enables you to be able to work offline and also prevents un intentional overwrites.

Some Usseful Git Commands.

To create a new local repository you would use "git init". To add one or more files to the staging area you would use "git add". "git commit" will commit changes to the head but not yet to the remote repository. To list the files you have changed and those you still need to add or commit you would use "git status". To fetch and merge changes on the remote server to your local working directory you would use "git pull".
