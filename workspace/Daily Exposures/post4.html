If you want to launch a simple website but dont want to jump through all the hoops of a hosting company or just looking for a cheaper alternative, GitHub Pages is your tool!

This post assumes that you have a Github account and some basic knowledge of the version control system Git.

Create Your Repository

Navigate over to GitHub and create a new repository for your website. Here are directions if you need help. The files that make up your website will need to be stored within this new repository.

Now open up a terminal (command prompt on Windows) and clone a copy of the repo you just made onto your computer.

// Retrieve a copy of your GitHub repo.
git clone https://github.com/user/repository.git
// Move into that directory.
cd repository

Make sure that you change the clone URL to the URL of your GitHub repo. This can be found on the main project page.

Once you got your local copy, move into the project folder using the cd command.

Create an Orphan Branch

Next it is time to create a new orphan branch within your repo that will hold all of your website files. This new branch should be called "gh-pages"

git checkout --orphan gh-pages

Adding Your Website Files

Now that your repo has been properly setup it's time to add all of the HTML, CSS and Javascript files that make up your website. There are multiple ways that you can go about doing this but I am going to only speak of the GitHub client.

Download and Install the Github client for Mac or PC.

After installing the client navigate to your cloned repo from the first step in a file explorer. Drag and drop it into your GitHub folder that you specified from installation.

Push Changes to GitHub

Open your GitHub client. Click the plus sign in the top left and add your local repository to your client.

After that you will see your new repo listed on the left. Select the new one and add a small description of the changes and hit commit.

Once you have selected commit, navigate to the top right corner and select Sync. This will upload your files to your GitHub Page site.

That’s it! Your website should now be available at

http://username.github.io/repository/.

Using a Custom Domain

Now that you have your files hosted on a GitHub server what if you want to change your web address from the username.github.io address? Thats where this part comes in and this is how I run my website for free.

First you will need to create a new file in your local Github repo file from earlier called

CNAME

Inside this file you will have your domain name that you wish to use. It should look something like this:

paulgovoni.com

Next you will need to update your DNS records for your domain name that you plan on using. This can usually be done through the provided domain name control panel from the domain registrar.

With your DNS provider, create A records that reslove to the following IP addresses:

192.30.252.153
192.30.252.154

It may take some time for the DNS changes to take effect. Usually no longer then a few hours. Once the changes have been applied, you should be able to access your new website from your custom domain name.

You Are Live!

Keep in mind that GitHub pages does not meet the needs of everyone. You are limited to using static assets (HTML, CSS and JS) for your websites. But if you are ready to launch your first live website for the world to see this is a very good point to start! Dont forget that its free too!
